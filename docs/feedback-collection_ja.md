# tennis-caliper-ar フィードバック収集ガイド

更新日: 2026-05-05

## 目的

知り合いテスターから「うまくいかなかった」情報を再現可能な形で回収する。

## 収集フロー

1. テスターにローカルプロトタイプを触ってもらう
2. 問題が出たら、画面下部の「フィードバック収集」で内容を入力
3. 「フィードバックを追加」で蓄積
4. 最後に「JSONを書き出し」でファイル保存
5. JSONを開発側に共有

## 最低限集める情報

- 期待した動作
- 実際の結果
- 再現手順
- 実行コンテキスト（入力ソース、誤差モード、環境プロファイル、コート種別、ユーザーエージェント）

## 運用ルール

- 1件1事象で記録する
- 「なんか変」ではなく再現手順を3ステップ以上書く
- 収集したJSONは日付付きで保管する
