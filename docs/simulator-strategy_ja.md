# tennis-caliper-ar シミュレータ戦略（日本語）

更新日: 2026-05-05

## 方針（結論）

テニスコート現地での検証機会が限られるため、開発の主軸はエミュレータ/シミュレータに置く。
現地/実機でしか確認できない項目は最後にまとめて検証する。

## 採用する検証レイヤー

### 1. WebXR APIエミュレータ

- 目的: WebXR API呼び出しと状態遷移の検証
- 対象:
  - `requestSession` フロー
  - XR座標処理
  - 入力イベント処理
  - エラーハンドリング
- 非対象:
  - 実カメラ品質
  - 実空間トラッキング精度

### 2. 保存動画シミュレータ（ダミー入力）

- 目的: カメラ入力がなくても描画・判定ロジックを再現可能にする
- 対象:
  - ガイド線重畳
  - キャリブレーションUI
  - 誤差モード判定
  - 警告表示分岐
- 非対象:
  - 実センサー追従品質

### 3. ヘッドレス/自動テスト

- 目的: 回帰防止
- 対象:
  - 境界値テスト
  - 状態遷移テスト
  - UI表示分岐テスト

## ローカル完了条件（シミュレータ段階）

1. 誤差モード（±1/±3/±5）判定が仕様どおり動作
2. 環境プロファイル/コート種別の警告分岐が再現可能
3. 保存動画入力でキャリブレーション〜判定まで一連動作
4. WebXR APIフローがエミュレータで破綻しない

## 最終的に実機が必要な項目

- 実カメラ露出変化・ノイズ下の安定性
- 実空間トラッキング追従性
- 端末個体差（iOS/Androidブラウザ差）

## 運用ルール

- 新機能はまずシミュレータで仕様確定し、その後に実機へ移す。
- ドキュメントは `docs/handoff_ja.md` と同時更新し、次チャットで再開可能にする。
