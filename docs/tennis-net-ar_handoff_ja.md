# テニスネット高さ調整ARアプリ 引き継ぎメモ（新チャット用）

更新日: 2026-05-05

## 1. 目的（ユーザー要望）

- スマホカメラでテニスコートのネットを映し、**ネット高さ調整**を支援したい。
- 画面上に実映像へ重ねて、**正しい高さのガイド線**を表示したい。
- ARの利用を想定。

## 2. この案は実現可能か（結論）

- 実現可能。
- ただし精度は方式次第:
  - 2Dオーバーレイのみ: 簡易（ズレやすい）
  - AR平面認識 + キャリブレーション: 実用的

## 3. 推奨技術スタック（初期提案）

- iOS優先なら: **ARKit / RealityKit**
- 両OSなら: **Unity + AR Foundation**（ARKit/ARCore）

## 4. MVP（最初の到達点）

1. カメラプレビュー表示
2. 基準点キャリブレーション（例: 支柱根元2点）
3. ガイド線オーバーレイ表示（中央 0.914m、ポスト側 1.07m）
4. 高低差の表示（高い/低い、可能ならcm）

## 5. 精度で先に決めるべきこと

- 許容誤差: ±1cm / ±3cm / ±5cm
- 使用条件: 屋外昼のみ or 夕方/夜も含む
- 想定端末: iPhone限定か、Android含むか

## 7. 対話チューニング（新チャットで明示推奨）

ユーザーとの直近対話から、以下を最初に伝えると進行が安定する:

- **公式ソース優先**: 仕様/要件は公式ドキュメントを確認してから回答
- **曖昧語を避ける**: 「可能性」より「確認方法と確定条件」を先に出す
- **実装前に妥当性判断**: 「本当に必要か」を短く評価してから作業
- **要点先出し**: 結論→根拠→次アクションの順
