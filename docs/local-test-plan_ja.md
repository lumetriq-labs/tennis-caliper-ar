# tennis-caliper-ar ローカルテスト計画（日本語）

更新日: 2026-05-05

## 目的

テニスコート現地検証の前に、ローカルで再現できる失敗を先に潰す。
外でしか確認できない項目を最小化して、現地検証の効率を上げる。

## 方針

- フェーズ1は「動作安定性」と「判定ロジックの妥当性」を優先する。
- 精度の絶対値（cm）は暫定扱いとし、まず判定の一貫性を確認する。
- 実データ不足はモック入力で補い、同じ入力で同じ結果が出ることを担保する。

## ローカルで先に検証する項目

1. カメラ権限フロー（許可/拒否/再試行）の表示分岐
2. キャリブレーション手順のUI遷移（開始/完了/再実行）
3. 許容誤差モード（±1/±3/±5cm）切替時の判定閾値反映
4. 環境条件不足時の警告表示（理由と次手順）
5. Webブラウザ差分（最低: iOS Safari, Android Chrome 相当）
6. 端末回転・画面再描画時の状態保持

## テスト入力（モック）

- 差分値サンプル: `-7, -4, -2, 0, +2, +4, +7` cm
- 環境プロファイル: `DAY_CLEAR`, `DAY_CLOUDY`, `NIGHT_COURT`, `INDOOR_LED`
- コート種別: ハード / クレイ / 人工芝

## 受け入れ条件（ローカル）

- 同一入力に対して判定結果が安定している。
- 閾値切替で判定が仕様どおり変化する。
- エラー時に曖昧語で終わらず、再試行手順が表示される。
- 主要ブラウザで致命的クラッシュがない。

## 現地検証へ進む条件

- ローカル受け入れ条件をすべて満たす。
- 未解決課題が「現地でしか確認できない項目」に限定される。
